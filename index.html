PC端/移动端适配方法(基于 Bootstrap 栅格系统):

适合两端样式差别不大的网站, 共用同一套代码, 只是通过 media query 做适配

  • 观察设计稿两端大概的样式区别, 例如移动端是两列布局, PC端是三列布局, 移动端某个内容隐藏等等
  • 移动先行(Mobile first): 就是先从做移动端(xs)的样式开始
  • PC端简单的以 @media (min-width: 768px) (sm)为断点
  • 涉及到栅格的时候灵活使用多套 .col-[xs/sm/md/lg]-* 和 .visible-* / .hidden-*
  • 尽量避免使用 px 单位, 特别是尽量避免限定(写死)容器内内容的宽度(这样多半是根据设计稿分辨率测量的值, 分辨率不一样时, 就会出现横向的滚动条), 多使用百分比和其他相对单位
  • 大部分的适配工作是给予两套(文字)大小或者位置调整

示例: 移动端只出现 2 个, PC端出现 4 个

col-xs-6 col-sm-3
col-xs-6 col-sm-3
hidden-xs col-sm-3
